Dockerfile

Docker может создавать изображения автоматически, читая инструкции из Dockerfile. Dockerfile - это текстовый документ, который содержит все команды, которые пользователь может вызвать в командной строке для сборки изображения. Используя сборку Docker, пользователи могут создавать автоматизированные сборки, которые последовательно выполняют несколько команд командной строки.

На этой странице описаны команды, которые вы можете использовать в Dockerfile. Когда вы закончите читать эту страницу, обратитесь к Руководству по Dockerfile для ознакомительного руководства.
использование

Команда docker build создает образ из Dockerfile и контекста. Контекст сборки - это набор файлов в указанном месте PATH или URL. PATH - это каталог в вашей локальной файловой системе. URL-адрес - это хранилище Git.

Контекст обрабатывается рекурсивно. Таким образом, PATH включает в себя любые подкаталоги, а URL включает в себя хранилище и его подмодули. В этом примере показана команда построения, которая использует текущий каталог в качестве контекста:
